Introduction to TCP/IP

TCP/IP (Transport Control Protocol / Internet Protocol) is the standard network stack used by the internet so that internet-connected hardware and software can communicate with each other. All modern hardware and software since the early 2000s supports TCP/IP by default including computers, smartphones, tablets, smart TVs and game consoles.

Having a working knowledge of TCP/IP and the common protocols assists with configuring network equipment and troubleshooting problems that can occur with it.

What is a protocol?

My old A-level computing textbook defined a protocol with the following: "A protocol is a set of rules governing the communication of two or more devices."

Many protocols are open, meaning they are publicly documented and can be used without requiring permission from the developer(s) of that protocol. Most open protocols have been defined with a Request For Comments (RFC) document to assist developers when they integrate that protocol with their software.

Most protocols utilise either TCP (Transport Control Protocol) or UDP (User Datagram Protocol) as transport protocol to send data as packets from source to destination. An analogy to remember these is to think of how a postal service will either send a package with tracking and a required signature by the recipient, or just send the package untracked instead.

  • TCP - sends packets and verify they have arrived intact, if they haven't data is re-sent. Slower than UDP, but neccessary for protocols that require data integrity such as e-mail and file transfers.
  • UDP - sends packets without verifying they have arrived intact, making it faster than TCP. Often used by protocols that stream data such as audio or video.

Post Office Protocol 3 (POP3)

Post Office Protocol 3 (POP3) is an open and standardised protocol used for receiving e-mails, that is defined by RFC. POP3 downloads the contents of the mailbox directly from the mail server without syncing any alterations made to the mailbox and for that reason it is not commonly used anymore. However, most e-mail clients still support it including Microsoft Outlook, Apple Mail and Mozilla Thunderbird.

Internet Message Access Protocol (IMAP)

Internet Message Access Protocol (IMAP) is another open and standardised protocol used for receiving e-mails. Unlike POP3, IMAP syncronises changes made with the mail server when it retrieves new e-mails. Alongside POP3, IMAP is supported by most e-mail clients including Microsoft Outlook, Apple Mail and Mozilla Thunderbird.

HyperText Transfer Protocol (HTTP)

HyperText Transfer Protocol (HTTP) is an open and standardised protocol used by web browsers for retrieving webpages written in HyperText Markup Language (HTML) from a web server. If protocol can't retrieve the file it will display an error, such as the famous - Error 404: Page Not Found.

HyperText Transfer Protocol Secure (HTTPS)

HyperText Transfer Protocol Secure (HTTPS) behaves indentically to HTTP, though it uses a certificate to provide asymmetrical-key encryption between the user's web browser and the web server the page is stored on. This encryption is neccessary to prevent man-in-the-middle As of 2021, most websites use HTTPS and this can be verified by viewing the certificate by clicking the padlock icon at left-side of the web browser's address bar.

Trivial File Transfer Protocol (TFTP)

Trivial File Transfer Protocol (TFTP) is a simple file transfer protocol that doesn't use authentication or security when transferring data, nor verify it has been received correctly. It is used by the Pre-boot eXecution Enviroment (PXE) when an operating system image is installed from a server on the Local Area Network (LAN).

Domain Naming Service (DNS)

Domain Naming Service (DNS) is an open and standardised protocol that converts a Fully Qualified Domain Name (FQDN) or hostname to an IP address.

Secure SHell (SSH)

Secure SHell (SSH) creates a secure tunnel using TLS between 2 computers for the purposes of running remote commands (using a shell such as GNU Bash) or sending data such as files (with the SSH File Transfer Protocol (SFTP)). It can also be used to secure Telnet, but is mainly used as an alternative to it.

Internet Control Message Protocol (ICMP)

Internet Control Message Protocol (ICMP) is used to test network connectivity between 2 devices. A common use of ICMP is the ping command which sends several packets while keeping track of the time it takes for them to be acknowledged.
